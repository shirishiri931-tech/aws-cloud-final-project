# חלק א' – מסמך אפיון

> מסמך זה נוצר אוטומטית באמצעות צינור ה‑`bootstrap` של docsync מתוך קוד המקור של
> המערכת (8 פונקציות Lambda ב‑Python, אפליקציית React, ותשתית AWS).

## על המערכת

**DocumentFlow Cloud** היא מערכת ענן ללא‑שרת (Serverless) לניהול מסמכי פרויקט,
הבנויה על גבי AWS. המערכת מספקת **מקור אמת יחיד** למסמכי הארגון ומנהלת אחריות,
היסטוריית גרסאות, מועדי יעד וסטטוס אישור — בליווי התראות דוא"ל בכל שלב.

| פרט | ערך |
|------|------|
| שם הפרויקט | DocumentFlow Cloud |
| קורס | ניהול מערכות ענן AWS – פרויקט גמר |
| מוסד | המכללה האקדמית להנדסה אזריאלי · 2025‑2026 |
| מרצה | אורי ברמן |
| אזור AWS | us-east-1 |
| ארכיטקטורה | Serverless (Lambda · API Gateway · DynamoDB · S3 · Cognito · SES · EventBridge) |

## תפקידי משתמש (Roles)

| תפקיד | הרשאות |
|--------|---------|
| **Admin** | מנהל משתמשים ושיוך תפקידים דרך לוח הניהול. יוצר משתמשים ב‑Cognito עם סיסמה זמנית אקראית (ללא סיסמאות קבועות). |
| **PMO** | יוצר מסמכים, משייך אחראי ומבקר, קובע מועדי יעד ועוקב אחר סטטוס כל המסמכים. |
| **Reviewer** | סוקר מסמכים שהוקצו לו, מוריד ומעלה גרסאות מתוקנות/מוערות, מאשר/דוחה/מבקש תיקון. |

אין תפקיד "Owner" קבוע. לכל מסמך שדה `assignedUserId`/`assignedUserEmail` המייצג את
האחראי הנוכחי; אותו אדם יכול להיות אחראי על מסמך אחד ומבקר על אחר.
